严重问题 (2)

严重 安全 src/main.py:61-62
debug_middleware 将完整请求头输出到日志，包含 Authorization 中的 API Key
风险 — API Key 以明文泄漏到日志输出/文件中，任何能读取日志的人都能获取密钥。Codex CLI 会携带其 API Key 发送请求，该 Key 随 headers 被全量记录。
修复 — 从日志中过滤敏感头字段（Authorization、X-API-Key 等）后再输出，或直接移除 headers 日志输出，改为仅记录 Content-Type、Content-Length 等元信息。
严重 安全 src/config.py:80-81
CLI_PROXY_API_BASE 环境变量直接覆盖 api_base，无任何 URL 校验
风险 — 若攻击者能控制环境变量，可将所有请求（携带 API Key）重定向到恶意服务器，造成 SSRF + 密钥泄漏。
修复 — 对 URL 进行 scheme/host 校验：只允许 https:// 协议，可配置白名单域名（如 api.deepseek.com 及其子域名）。
